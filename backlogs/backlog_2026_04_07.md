## Team Coding & Git Standards
(Chosen from Mozilla guidelines)

**Core Rules** (full rulset [here](https://github.com/Bryce-W-Frazier/cs-F372-project/blob/main/codingStandards.md))
- Indents are exactly 2 spaces. Functions cannot be over 30 lines.
- Naming: `camelCase` for functions, `snake_case` for local variables, `UPPER_SNAKE_CASE` for globals.
- Arrays: Always declare with `[]` and add with `.push()`.
- Loops: Use `for...of` or `.forEach()`. Avoid old-school `for (;;)` loops.
- Error Handling: MUST use `try/catch` blocks for network/database calls.
- Comments: Every file needs a 3-line header (filename, purpose, start date). Only use `//` for comments, never `/* */`. Don't let comment lines get too long.

**Git / Workflow Rules:**
- Message the Discord group before working, and push your commits after every session.
- Commit format: `<Type>: <Description>` (Example: `Feat: Updated index.html`).
- ALWAYS run `git pull --rebase` to avoid messy merges.
